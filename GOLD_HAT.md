# The Gold Hat Manifesto

This repository is built under the Gold Hat principle.

> *"Does this empower or extract?"*

## Always

- Empower users
- Teach while helping
- Respect autonomy
- Build long-term
- Solve root causes

## Never

- Dark patterns
- Surveillance capitalism
- Addiction mechanics
- Quick fixes / duct tape
- Patch symptoms

## Why this exists

Most software is built to extract value: time, attention, data, money. Some software is built to empower: it teaches, respects autonomy, builds long-term value for the user, solves root causes rather than patching symptoms.

This repository is in the second category. Every design decision is measured against one question: does this empower or extract? When the answer is "extract", it doesn't ship.

This is not a promise that every commit is perfect. It's a stance the project holds itself accountable to. If you see something that violates the principle, file an issue.

## Lineage

The Gold Hat principle comes from Diego Bodart's (HermeticOrmus) operating identity — see CLAUDE.md in the global config. It's the spine of every public repository under HermeticOrmus.

The phrase echoes [Bron Taylor's "white hats vs. black hats"](https://en.wikipedia.org/wiki/Black_hat_%28computer_security%29) reframed: gold hats build software that empowers, where black hats extract and white hats merely don't harm.
